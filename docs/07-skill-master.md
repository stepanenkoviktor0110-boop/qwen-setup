# Skill Master — управление навыками

Skill Master берёт накопленные паттерны из Quick Learning и встраивает их в навыки агента. Вместо того чтобы просто хранить уроки в списке, они становятся частью инструкций — агент учитывает их автоматически.

---

## Как это работает

1. **Проверка** — считает, сколько паттернов ещё не обработано. Если меньше 25 — рано, выходит.

2. **Загрузка** — собирает необработанные паттерны, группирует по категориям.

3. **Анализ** — для каждого навыка проверяет: есть ли уже правило, покрывающее этот паттерн?
   - **Нет покрытия** → добавляет правило автоматически
   - **Частичное покрытие** → предлагает расширение и спрашивает тебя
   - **Полное покрытие** → пропускает

4. **Очистка** — удаляет обработанные паттерны из reasoning-patterns.md, обновляет индекс.

---

## Когда запускать

Quick Learning подскажет — когда накопится 25+ необработанных паттернов, ты увидишь сообщение:

```
Накопилось N необработанных триад — запусти /skill-master.
```

Или запусти вручную:

```
skill master
```

---

## Что происходит при запуске

Skill Master покажет результаты в таком формате:

```
Skill Master: обработано 28 триад.
Применено автоматически: 20 правил → methodology
Споров решено: 5/5
Пропущено (уже покрыто): 3
```

Если есть спорные случаи — покажет каждый по очереди:

```
Dispute: "category default bias" → methodology

Триада: решение кажется очевидным → проверить данные → избежать предвзятости

Существующее правило:
  "Всегда проверяй допущения перед реализацией"

Предлагаю расширить до:
  "Всегда проверяй допущения. Особенно когда решение кажется очевидным из категории — конкретный случай может отличаться"

Применить? [да / нет / пропустить]
```

## Установка

Если запускал `setup-qwen.sh` — уже установлен.

Вручную:

```bash
mkdir -p ~/.qwen/skills/skill-master
cp skills/skill-master/SKILL.md ~/.qwen/skills/skill-master/
```

---

## Добавление своих навыков

По мере роста проекта можно добавлять новые навыки в `~/.qwen/skills/`. Каждый навык — папка с файлом `SKILL.md`:

```
~/.qwen/skills/
├── methodology/SKILL.md
├── quick-learning/SKILL.md
├── skill-master/SKILL.md
└── мой-навык/SKILL.md          # твой кастомный навык
```

Формат SKILL.md:

```yaml
---
name: мой-навык
description: Описание — когда использовать этот навык
---

# Инструкции

Шаги, правила, примеры...
```

Обнови таблицу маппинга в Skill Master (`Category → Skill Mapping`), чтобы паттерны из Quick Learning попадали в нужные навыки.
