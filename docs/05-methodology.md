# Методология AI-разработки

Структурный подход к работе с AI-агентом. Вместо хаотичных запросов — чёткий пайплайн: идея → описание → архитектура → задачи → код → документация.

---

## Зачем это нужно

Без методологии агент:
- Забывает контекст между сессиями
- Пишет код без плана и потом переделывает
- Не знает, что уже сделано и что осталось
- Теряет решения, которые принимались по ходу

С методологией:
- Знания о проекте хранятся в файлах и доступны всегда
- Каждая фича проходит одинаковый путь
- Решения фиксируются и не теряются

---

## Пайплайн

```
Идея → User Spec → Tech Spec → Задачи → Код → Готово
```

### 1. User Spec — что и зачем

Агент проводит интервью: задаёт вопросы, уточняет детали, фиксирует требования.

Результат: файл `user-spec.md` — описание фичи на понятном языке. Что делаем, для кого, какие сценарии.

### 2. Tech Spec — как

На основе user-spec агент изучает код проекта и предлагает техническое решение: архитектура, зависимости, план тестирования.

Результат: файл `tech-spec.md` с архитектурой и списком задач.

### 3. Задачи — декомпозиция

Каждая задача из tech-spec становится отдельным файлом с чёткими критериями приёмки: что должно работать, какие файлы менять, от каких задач зависит.

Результат: файлы `tasks/1.md`, `tasks/2.md` и т.д.

### 4. Код — реализация

Агент берёт задачу, читает контекст, пишет тест → код → проверяет. После каждой задачи — коммит.

### 5. Готово — финализация

Обновляем документацию проекта, архивируем фичу.

## Структура проекта

Каждый проект использует одинаковую структуру папок:

```
проект/
├── .qwen/
│   └── skills/
│       └── project-knowledge/
│           └── references/
│               ├── project.md       # О проекте
│               ├── architecture.md  # Стек и архитектура
│               ├── patterns.md      # Конвенции кода
│               └── deployment.md    # Деплой и мониторинг
├── work/
│   ├── feature-name/
│   │   ├── user-spec.md
│   │   ├── tech-spec.md
│   │   ├── decisions.md
│   │   └── tasks/
│   └── completed/                   # Архив завершённых
└── QWEN.md                          # Инструкции агенту
```

---

## Project Knowledge — база знаний

4 файла описывают весь проект. Агент читает их в начале сессии.

| Файл | Содержит |
|------|----------|
| `project.md` | Что делает проект, для кого, ключевые фичи |
| `architecture.md` | Стек, структура папок, зависимости, модель данных |
| `patterns.md` | Git workflow, стиль кода, тестирование |
| `deployment.md` | Платформа, переменные окружения, мониторинг |

**QWEN.md** содержит только минимум: название проекта и ссылку на project-knowledge. Вся информация — в файлах базы знаний.

---

## Decisions.md — журнал решений

Во время реализации агент записывает каждое решение:
- Что решили
- Почему (какие альтернативы рассматривали)
- Что изменилось от плана

Это помогает при возврате к проекту — видно, почему всё сделано именно так.

## Установка навыка

Если запускал `setup-qwen.sh` — навык `methodology` уже установлен в `~/.qwen/skills/methodology/`.

Вручную:

```bash
mkdir -p ~/.qwen/skills/methodology
cp skills/methodology/SKILL.md ~/.qwen/skills/methodology/
```

Агент автоматически подхватит навык и будет использовать при работе.
